# Journey English Version 0.6 Plan

## Project Goal

Upgrade Journey English from an English learning tool into an English Adventure Companion that children want to come back to every day.

Version 0.6 focuses on curriculum and experience planning based on real family beta feedback, without introducing new AI-driven behavior.

## Objectives

- Make the product feel like a guided adventure instead of a sequence of isolated lessons.
- Improve daily continuity so learners can naturally continue from one day to the next.
- Strengthen curriculum structure for longer use across a 30-day journey.
- Prepare a clearer product path for future companion, speaking, and parent-facing features.
- Keep the system local-first, simple, and stable while validating the next product direction.

## Version Scope

Version 0.6 is a planning release only.

Included in scope:

- Curriculum V2 planning
- Learning-experience direction
- Version roadmap update
- Review of family beta feedback
- Acceptance criteria for future implementation work

## Out of Scope

- No new product features
- No UI redesign
- No component changes
- No database changes
- No AI integration work
- No speech scoring
- No review engine redesign
- No parent dashboard implementation

## Milestones

### Milestone 1: Feedback Consolidation

- Summarize family beta feedback from Version 0.5
- Separate positives, problems, and improvement requests
- Identify P0 and P1 experience priorities

### Milestone 2: Curriculum V2 Definition

- Define the role of Journey English as an adventure companion
- Clarify how a 30-day learning path should feel across repeated use
- Define learning flow expectations for daily adventures, review, celebration, and re-entry

### Milestone 3: Roadmap Alignment

- Align Version 0.6 through Version 1.0
- Clarify which problems belong to curriculum, AI companion, speaking, and parent dashboards
- Ensure future milestones build on validated user behavior instead of assumptions

### Milestone 4: Implementation Readiness

- Produce planning documents that can be reviewed before coding starts
- Define acceptance criteria for Version 0.6 implementation work
- Keep Version 0.5 stable during planning

## Risks

- Family beta feedback may be directional but still too narrow if it comes from a small number of learners.
- Curriculum expansion can create quantity without improving daily engagement if adventure pacing is not redesigned carefully.
- Future AI work can distract from the current experience goal if scope boundaries are not kept strict.
- Additional features can increase complexity faster than children or parents can understand the product.
- If Version 0.6 planning is vague, later milestones will drift back into feature-first delivery instead of experience-first delivery.

## Acceptance Criteria

Version 0.6 planning is complete when:

- The product goal for Version 0.6 is documented clearly.
- Scope and out-of-scope boundaries are explicit.
- Family beta feedback is captured in a reusable written summary.
- The roadmap from Version 0.6 to Version 1.0 is documented.
- Future implementation work can start from these documents without redefining the product direction.
- No product logic, UI, or database behavior changes are introduced in this planning PR.
